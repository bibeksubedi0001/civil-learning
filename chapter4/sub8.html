Semantic Segmentation

From bounding boxes to pixel-perfect masks — classifying every single pixel in an image to quantify crack area, map structural damage, and analyse road surfaces at unprecedented granularity using encoder-decoder architectures, U-Net skip connections, and specialised loss functions.

55 min read 2 interactive demos 4 quiz questions

What Is Semantic Segmentation?

In the previous sub-chapter you saw how object detection draws bounding boxes around objects. But a rectangle is a coarse approximation — it includes background pixels and misses irregular shapes. Semantic segmentation takes visual understanding to pixel-level precision: every single pixel in the image is assigned a class label. There are no boxes, no anchors — just a dense prediction map with the same spatial resolution as the input.

Formally, given an input image of size H × W × 3, a semantic segmentation model produces an output tensor of size H × W × C, where C is the number of classes. Each spatial position (i, j) contains a probability distribution over classes, and the argmax yields the predicted class for that pixel. The result is a segmentation mask — a colour-coded map where each colour represents a different class.

Semantic vs. Instance vs. Panoptic

It is important to distinguish between three related but different segmentation paradigms:

  1. Semantic Segmentation — every pixel gets a class label, but individual instances of the same class are not distinguished. Two adjacent cracks are merged into one "crack" region.
  2. Instance Segmentation — each individual object gets its own mask. Two adjacent cracks are separated as "crack #1" and "crack #2." Combines detection with pixel masks (e.g., Mask R-CNN).
  3. Panoptic Segmentation — unifies semantic and instance segmentation. "Stuff" classes (sky, road, wall) get semantic labels; "thing" classes (cars, people, cracks) get instance-level masks.

For most civil engineering inspection tasks — crack area measurement, road surface classification, corrosion mapping — semantic segmentation is the natural starting point because we care about the total area of each defect type, not necessarily individual counts.

Civil Engineering Analogy

Imagine a topographic survey map. Classification tells you "this site has a river." Detection draws a rectangle around the approximate river area. Semantic segmentation is the full contour map — every square metre of land is labelled as river, forest, rock, or soil. You can compute exact areas, perimeters, and boundaries. That is the power of pixel-level classification for structural inspections: instead of "crack detected," you get "the crack occupies 2.4% of the beam surface, with a maximum width of 0.8 mm and a total length of 1.3 m."

Why Not Just Use Bounding Boxes?

Bounding boxes work well for compact, roughly rectangular objects. But structural defects are rarely rectangular:

  • Cracks are thin, branching, and irregular. A bounding box around a diagonal crack is mostly background.
  • Spalling regions have jagged boundaries that boxes cannot capture.
  • Corrosion stains spread in amorphous patterns that defy rectangular approximation.
  • Road distress — potholes, rutting, alligator cracking — requires area measurements, not just locations.

Semantic segmentation gives you the exact pixel footprint of each defect type, enabling quantitative measurements (area, width, length) that feed directly into maintenance decision models and structural health monitoring dashboards.

Fully Convolutional Networks (FCN)

The Seminal Idea

The breakthrough paper by Long, Shelhamer, and Darrell (2015) introduced Fully Convolutional Networks (FCN) by making a deceptively simple observation: you can replace the fully-connected (dense) layers at the end of a classification CNN with 1×1 convolutional layers. The result is a network that accepts images of any size and outputs a spatial map of class predictions rather than a single class vector.

In a standard classification CNN like VGG-16, the feature maps are progressively down-sampled by pooling layers (e.g., from 224 × 224 down to 7 × 7), and then flattened into a 1D vector for the final dense layers. FCN keeps everything spatial. Instead of flattening, it applies 1×1 convolutions that act as per-pixel classifiers, producing a coarse prediction map at 7 × 7 resolution.

The Upsampling Problem

A 7 × 7 output is far too coarse for pixel-level predictions on a 224 × 224 input. The spatial resolution has been destroyed by 32× downsampling across five pooling stages. FCN addresses this with upsampling (also called deconvolution or transposed convolution) to recover the original resolution.

The paper introduced three variants with progressively finer outputs:

Variant Stride Skip Connections Output Quality
FCN-32s 32× None Very coarse, blobby boundaries
FCN-16s 16× pool4 Better edges, still imprecise
FCN-8s 8× pool3 + pool4 Sharp boundaries, best quality

The key insight behind FCN-8s was skip connections: combining the coarse, semantically rich features from deep layers with the fine-grained, spatially precise features from earlier layers. This idea would become the cornerstone of the next major architecture — U-Net.

Transposed Convolution (Deconvolution)

How does a network increase spatial resolution? A regular convolution with stride 2 halves the spatial dimensions. A transposed convolution (sometimes misleadingly called "deconvolution") does the reverse: it inserts zeros between input elements, effectively mapping a smaller feature map to a larger one.

Output Size (Transposed Conv)
H_out = (H_in - 1) × stride - 2 × padding + kernel_size

For example, a transposed convolution with kernel=4, stride=2, padding=1 exactly doubles the spatial dimensions — the inverse of a kernel=4, stride=2, padding=1 regular convolution. The kernel weights are learned during training, so the network discovers the best way to upsample feature maps for each task.

Alternative Upsampling Methods

Transposed convolution is not the only way to upsample. Other strategies are commonly used:

  • Bilinear interpolation — a fixed (non-learned) method that resizes the feature map using weighted averages of neighbouring values. Fast but limited.
  • Nearest-neighbour interpolation — duplicates each value to fill the larger grid. Simple, produces blocky results.
  • Sub-pixel convolution (PixelShuffle) — rearranges channels into spatial dimensions. Popular in super-resolution networks.
  • Bilinear + Conv — upsample with bilinear interpolation, then refine with a regular convolution. Avoids the checkerboard artifacts that transposed convolutions can produce.

The Encoder-Decoder Architecture

FCN demonstrated that segmentation is possible, but its architecture was a retrofit of a classification network. Modern segmentation models are designed from the ground up with a symmetric encoder-decoder structure:

Encoder (Contracting Path)

The encoder is a standard CNN backbone (e.g., ResNet, VGG, EfficientNet) that progressively downsamples the input through convolutions and pooling. At each stage, the spatial resolution halves while the number of feature channels doubles. The encoder captures what is in the image — high-level semantic information — but loses where it is.

Decoder (Expanding Path)

The decoder mirrors the encoder, progressively upsampling through transposed convolutions or bilinear interpolation. At each stage, the spatial resolution doubles and the number of channels halves. The decoder recovers where things are, reconstructing a full-resolution segmentation mask.

The critical question is: how does the decoder recover the fine spatial details that the encoder discarded during downsampling? The answer is skip connections.

The Information Bottleneck

At the bottom of the U-shape (the "bridge" or "bottleneck"), the feature map is at its smallest spatial resolution but highest channel depth. This is where the network holds its most abstract, compressed representation of the entire image. For a 256 × 256 input with four downsampling stages, the bottleneck might be 16 × 16 × 1024.

Without skip connections, the decoder would have to reconstruct all spatial detail from this compressed bottleneck — an impossible task for fine structures like hairline cracks. Skip connections solve this by providing a "shortcut" for spatial information to flow directly from the encoder to the corresponding decoder level.

Civil Engineering Analogy

Think of the encoder-decoder like writing a structural inspection report. The encoder is you walking through the structure, progressively summarising: "beam" becomes "reinforced concrete beam with flexural cracks." By the end you have a high-level summary (the bottleneck) but have lost the exact GPS coordinates of each crack. The decoder writes the detailed report, and the skip connections are your field notes and photographs from each floor — they let you recover the precise locations that the summary alone could not reconstruct.

U-Net: The Workhorse of Segmentation

Architecture Overview

U-Net (Ronneberger et al., 2015) was originally designed for biomedical image segmentation, where labelled data is scarce. Its elegant symmetric architecture, combined with dense skip connections, made it the de facto standard for any domain where precise pixel-level segmentation is required — including civil engineering inspection.

The name "U-Net" comes from the U-shape of the architecture when drawn as a diagram: the encoder forms the left descending arm, the bottleneck is the bottom, and the decoder forms the right ascending arm. Horizontal arrows connecting corresponding encoder and decoder levels represent skip connections.

U-Net Building Blocks

Each level of the U-Net consists of a convolutional block: two consecutive 3 × 3 convolutions, each followed by batch normalisation and a ReLU activation. The original paper used "valid" convolutions (no padding), which shrink the feature map slightly at each step. Modern implementations typically use "same" padding to preserve spatial dimensions within each block.

  1. Encoder Block — Conv 3×3, BN, ReLU → Conv 3×3, BN, ReLU → Max Pool 2×2 (halves spatial resolution, doubles channels).
  2. Bottleneck Block — Conv 3×3, BN, ReLU → Conv 3×3, BN, ReLU (no pooling, maximum channel depth).
  3. Decoder Block — Transposed Conv 2×2 (doubles spatial resolution, halves channels) → Concatenate with encoder skip → Conv 3×3, BN, ReLU → Conv 3×3, BN, ReLU.
  4. Final Layer — Conv 1×1 mapping the final feature channels to C output classes. Softmax (or sigmoid for binary) activation produces per-pixel probabilities.

Skip Connections in Detail

The defining feature of U-Net is the concatenation-based skip connection. At each decoder level, the upsampled feature map from the level below is concatenated (along the channel dimension) with the feature map from the corresponding encoder level. This doubles the number of channels entering the decoder's convolutional block, but the subsequent convolutions reduce it back.

Why concatenation rather than addition (as in ResNet)? Concatenation preserves both the high-resolution spatial features from the encoder and the semantically rich features from the decoder, letting the convolutional layers learn how to combine them optimally. Addition forces the features to share the same channel space, which can lose information.

Skip Connection (Concatenation)
decoder_input = concat(upsample(deeper_level), encoder_level)

For a 4-level U-Net with an input of 256 × 256 × 3:

Level Encoder Output Decoder Input (after concat)
Level 1 256 × 256 × 64 256 × 256 × 128
Level 2 128 × 128 × 128 128 × 128 × 256
Level 3 64 × 64 × 256 64 × 64 × 512
Level 4 32 × 32 × 512 32 × 32 × 1024
Bottleneck 16 × 16 × 1024

U-Net Variants

The original U-Net spawned many variants optimised for different tasks:

  • Attention U-Net — adds attention gates before skip connections to suppress irrelevant features, improving performance on small or sparse objects (e.g., hairline cracks).
  • U-Net++ — introduces dense nested skip connections, bridging the semantic gap between encoder and decoder features at each level.
  • ResUNet — replaces the convolutional blocks with residual blocks for deeper architectures without degradation.
  • 3D U-Net — extends to volumetric data, useful for CT scans of concrete cores or 3D point cloud segmentation.

Loss Functions for Segmentation

Choosing the right loss function is critical for segmentation because classes are almost always imbalanced. In a bridge inspection image, the crack pixels might comprise only 0.5% of the total — the remaining 99.5% is intact concrete. A naive cross-entropy loss would achieve 99.5% accuracy by predicting "no crack" everywhere.

Pixel-wise Cross-Entropy Loss

The most basic segmentation loss treats each pixel as an independent classification problem. For C classes:

Pixel-wise Cross-Entropy
L_CE = - (1 / N) ∑ ∑ y(i,j,c) · log(p(i,j,c))

Where N is the total number of pixels, y is the one-hot ground truth, and p is the predicted probability. This can be improved with class weights that penalise misclassification of rare classes more heavily:

Weighted Cross-Entropy
L_WCE = - (1 / N) ∑ w_c · y(i,j,c) · log(p(i,j,c))

Dice Loss

Dice loss directly optimises the Dice coefficient (also known as the F1 score), which measures the overlap between the predicted segmentation and the ground truth. It is inherently robust to class imbalance because it measures relative overlap rather than absolute pixel counts.

Dice Coefficient
Dice = 2 · |P ∩ G| / (|P| + |G|)
Dice Loss
L_Dice = 1 - (2 · ∑ p_i · g_i + ε) / (∑ p_i + ∑ g_i + ε)

The smooth factor ε (typically 1e-6) prevents division by zero when both prediction and ground truth are empty (no positive pixels). In practice, many practitioners use a combined loss:

Combined Loss
L = α · L_CE + (1 - α) · L_Dice

This combines the stable gradients of cross-entropy with the overlap sensitivity of Dice loss. A typical value is α = 0.5.

Focal Loss

Originally proposed for object detection (Lin et al., 2017), focal loss down-weights easy examples (background pixels correctly classified with high confidence) so the model focuses on hard examples (boundary pixels, thin cracks):

Focal Loss
L_focal = - α · (1 - p_t)^γ · log(p_t)

The focusing parameter γ (typically 2) controls how much easy examples are down-weighted. When γ = 0, focal loss reduces to standard cross-entropy.

Civil Engineering Analogy

Loss functions are like grading criteria for an intern's inspection report. Cross-entropy penalises every mistake equally — missing a crack is the same as mislabelling a clean area. Dice loss asks: "what fraction of the actual cracks did you correctly identify?" — it cares about coverage, not about how much clean concrete you correctly labelled. Focal loss is like a mentor who says: "stop spending time on obvious intact areas and focus on the ambiguous boundary regions."

Evaluation Metrics: IoU Per Class

Accuracy is a poor metric for segmentation due to class imbalance. Instead, the community uses Intersection over Union (IoU), also called the Jaccard Index, computed per class:

Intersection over Union (IoU)
IoU_c = TP_c / (TP_c + FP_c + FN_c)

Where TP is true positives (pixels correctly predicted as class c), FP is false positives, and FN is false negatives, all for class c.

Mean IoU (mIoU)

The mean IoU averages IoU across all classes:

Mean IoU
mIoU = (1 / C) ∑ IoU_c

mIoU is the standard benchmark metric for semantic segmentation (used in Pascal VOC, Cityscapes, ADE20K). For a civil engineering crack segmentation model, you might report:

Class IoU Interpretation
Background 0.987 Nearly perfect — background dominates
Crack 0.724 Good overlap with thin crack regions
Spalling 0.681 Reasonable; irregular boundaries are challenging
Corrosion 0.643 Harder due to colour overlap with stains
mIoU 0.759 Average across all four classes

Other Metrics

  • Pixel Accuracy — fraction of correctly classified pixels. Simple but misleading with imbalanced classes.
  • Dice Coefficient / F1 Score — closely related to IoU: Dice = 2 · IoU / (1 + IoU). Reports the same information in a different scale.
  • Boundary F1 (BF1) — evaluates segmentation quality specifically along object boundaries, critical for crack width measurements.
  • Frequency-Weighted IoU — weights each class IoU by its pixel frequency. Gives less weight to rare classes.

Civil Engineering Applications

Semantic segmentation is transforming how civil engineers measure, monitor, and maintain infrastructure. Here are the most impactful applications:

Crack Segmentation

Pixel-level crack masks enable quantitative measurement of crack area, width, length, and branching patterns. A U-Net trained on concrete surface images can detect cracks as thin as 0.1 mm from high-resolution photographs, computing crack density (mm²/m²) automatically.

Damage Mapping

After earthquakes or extreme events, UAV imagery is processed through segmentation models to create damage maps that classify each pixel as intact, cracked, spalled, collapsed, or debris. These maps guide emergency response and prioritise repair resources across large portfolios.

Road Surface Analysis

Pavement management systems use segmentation to classify road distress types: alligator cracking, longitudinal cracks, potholes, rutting, and patching. The segmented area directly feeds into Pavement Condition Index (PCI) calculations and maintenance scheduling algorithms.

Material Classification

Segmentation can identify and map different construction materials (concrete, steel, wood, masonry) in building facades from street- level imagery. This supports rapid inventory of building typologies for seismic vulnerability assessments at city scale.

Crack Segmentation in Practice

From Pixels to Measurements

Once you have a binary crack mask, converting pixels to physical measurements requires a calibration step. If you know the ground sampling distance (GSD) — the real-world size of one pixel — you can compute:

  • Crack area = (number of crack pixels) × GSD²
  • Crack width = measured perpendicular to the crack centreline using distance transform or skeletonisation.
  • Crack length = length of the skeletonised centreline × GSD.
  • Crack density = total crack area / inspection region area.

The typical pipeline for automated crack measurement is:

  1. Image Acquisition — capture high-resolution images from a fixed distance (tripod, UAV, or robotic crawler) to ensure consistent GSD.
  2. Segmentation — run the U-Net model to produce a binary crack mask (crack vs. background) or multi-class mask (crack, spalling, corrosion, efflorescence, intact).
  3. Post-processing — apply morphological operations (opening, closing) to remove noise and fill small gaps in the predicted mask.
  4. Skeletonisation — thin the crack mask to a 1-pixel-wide centreline using Zhang-Suen or medial axis transform.
  5. Measurement — compute width at regular intervals along the skeleton, total length, branching count, and area from the cleaned mask.
  6. Reporting — overlay the segmentation mask on the original image and generate quantitative metrics for the inspection database.

Damage Mapping After Extreme Events

Following earthquakes, hurricanes, or floods, rapid damage assessment saves lives and directs resources. Semantic segmentation applied to UAV or satellite imagery can classify structural damage at multiple scales:

  • Building level — classifying each building footprint as intact, minor damage, major damage, or collapsed using overhead imagery.
  • Component level — segmenting individual structural elements (columns, beams, walls, slabs) and their damage states from close-range photographs.
  • Infrastructure level — mapping road blockages, bridge damage, and utility disruptions across a city.

The segmentation masks are georeferenced and overlaid on GIS maps to create damage heat maps that decision-makers can use within hours of an event — compared to days or weeks for traditional field surveys.

Beyond U-Net: Modern Architectures

While U-Net remains the most popular architecture for domain-specific segmentation, several other architectures are worth understanding:

DeepLab Family

Atrous (Dilated) Convolution

DeepLab (Chen et al., 2017) introduced atrous spatial pyramid pooling (ASPP), which applies dilated convolutions at multiple rates to capture multi-scale context without downsampling. A dilated convolution with rate r inserts r - 1 zeros between kernel elements, effectively increasing the receptive field without adding parameters or losing resolution.

DeepLabv3+ combines ASPP with an encoder-decoder structure, achieving state-of-the-art results on Cityscapes and Pascal VOC. It is particularly effective for large-scale scene parsing where multi-scale context is critical.

SegFormer and Transformer-Based Approaches

Vision Transformers have entered the segmentation arena. SegFormer (Xie et al., 2021) uses a hierarchical Transformer encoder with overlapping patch embeddings and a lightweight all-MLP decoder. It achieves competitive results with fewer parameters and better generalisation to unseen domains.

Architecture Comparison

Architecture Key Innovation Best For Typical mIoU

Civil Engineering Analogy

Choosing a segmentation architecture is like choosing a structural analysis method. FCN is hand calculation — foundational, educational, but rarely used in practice. U-Net is a reliable finite element model — works great for most problems, especially with limited data. DeepLabv3+ is a sophisticated nonlinear FE analysis — handles complex multi-scale problems. SegFormer is the latest AI-powered generative design tool — cutting-edge, general-purpose, but requires understanding of newer paradigms.

Practical Tips for Segmentation Projects

Whether you are segmenting cracks on a bridge deck or road distress from a pavement survey vehicle, these practical guidelines apply:

Data Annotation

  • Annotation quality dominates model quality. A smaller dataset with precise pixel annotations beats a larger dataset with sloppy labels. Invest in clear annotation guidelines and inter-annotator agreement checks.
  • Use polygon tools, not brushes, for irregular shapes. Polygon annotation (e.g., in CVAT, LabelMe, or Supervisely) produces cleaner boundaries than freehand brushing.
  • Define boundary protocols. Does a crack include the damage halo around it? Where does spalling end and intact concrete begin? Ambiguity here directly translates to model uncertainty.

Training Strategy

  • Start with a pre-trained encoder. Use ImageNet pre-trained weights for the encoder (ResNet-34 or EfficientNet-B4) and train only the decoder initially. Fine-tune the full model with a lower learning rate.
  • Use heavy augmentation. Random rotations, flips, elastic deformation, brightness/contrast jitter, and Gaussian noise are essential when training data is limited (which it almost always is in CE applications).
  • Combine Dice + CE loss. The combined loss gives more stable training than either alone, especially for thin structures like cracks.
  • Monitor per-class IoU, not just mIoU. A high mIoU can mask poor performance on the classes you care about most (e.g., crack IoU of 0.3 hidden by background IoU of 0.99).

Deployment Considerations

  • Patch-based inference. For high-resolution images (4000 × 6000 from a DSLR), process overlapping patches and stitch the predictions. Use Gaussian weighting to blend overlapping regions and avoid edge artifacts.
  • Test-time augmentation (TTA). Run inference on flipped and rotated versions of the same image and average the predictions. Improves IoU by 1–3% at the cost of 4–8× inference time.
  • Morphological post-processing. Apply connected component analysis to remove small spurious regions. Apply dilation followed by erosion (closing) to fill gaps in thin crack predictions.

Common Pitfalls

Data leakage in tiled datasets: if you split a large image into patches for training and testing, patches from the same image must all go to the same split. Otherwise, the model memorises the texture of specific structures rather than learning generalisable crack features. Always split at the image level, not the patch level.

Civil Engineering Application: Pixel-Level Damage Mapping

Semantic segmentation classifies every pixel — enabling precise damage quantification that bounding boxes can't provide:

U-Net for Crack Segmentation

U-Net's encoder-decoder with skip connections excels at thin crack segmentation. With 256×256 inputs at 0.5mm/pixel, it segments crack widths down to 0.2mm. Output: a binary mask where each pixel is labeled crack (1) or intact (0). Damage area = pixel_count × pixel_area.

Pavement Condition Index

ASTM D6433 defines the Pavement Condition Index (PCI, 0-100) based on distress type and area. Segmentation CNNs automate this: segment all distress types → compute affected area per type → apply ASTM deduction curves → calculate PCI. Accuracy within ±5 points of manual surveys.

Flood & Erosion Mapping

Satellite image segmentation delineates flood extents, erosion zones, and land use changes. DeepLab v3+ segments water bodies at 10m resolution from Sentinel-2 imagery. Civil engineers use this for floodplain mapping, watershed delineation, and post-disaster damage assessment.
