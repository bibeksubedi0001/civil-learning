Training CNNs

From random weights to reliable predictions — the iterative process of forward passes, loss computation, backpropagation, and weight updates that teaches a CNN to distinguish cracks from intact concrete.

40 min read 2 interactive demos 4 quiz questions

Teaching a Machine to See Cracks

In the previous sub-chapter you learned how to design a CNN architecture — the structural blueprint. But an untrained network is like a newly built testing machine that has never been calibrated: the hardware is in place, yet every reading is meaningless. Training is the calibration process that adjusts the network's millions of parameters until its predictions match reality.

This sub-chapter walks you through every stage of the training pipeline: the forward pass that generates predictions, the loss function that quantifies errors, backpropagation that computes gradients, and the optimizer that updates weights. You will also learn practical techniques that civil engineers rely on — data augmentation, transfer learning, learning rate scheduling, and batch size selection — with interactive demos that bring these concepts to life.

Training a neural network is fundamentally an optimization problem. We seek the set of weights that minimizes the difference between the network's predictions and the ground-truth labels. The process is iterative: present data, measure error, compute corrections, apply corrections, and repeat — thousands or millions of times.

For civil engineering applications such as crack detection, corrosion assessment, and structural damage classification, training is where domain expertise meets machine learning. The choice of augmentation strategies, pretrained models, and hyperparameters directly determines whether a model achieves 70% accuracy (unusable) or 97% accuracy (deployment-ready).

Prerequisites

This sub-chapter assumes familiarity with CNN architectures (Sub-chapter 4.4), including convolutional layers, pooling, activation functions, and fully connected layers. Basic understanding of derivatives and gradients is helpful but not essential — we use visual analogies throughout.

The Training Loop: Four Repeating Steps

Every CNN training run follows the same four-step cycle, repeated for each batch of data across multiple epochs (full passes through the dataset):

The Four Steps

  1. Forward Pass — Feed a batch of images through the network, layer by layer, to produce predictions.
  2. Loss Computation — Compare the predictions to the true labels using a loss function (e.g., cross-entropy) to get a single scalar error value.
  3. Backpropagation — Compute the gradient of the loss with respect to every weight in the network by applying the chain rule of calculus backwards through the layers.
  4. Weight Update — Use an optimizer (e.g., Adam, SGD) to adjust each weight in the direction that reduces the loss, scaled by the learning rate.

After processing all batches in one epoch, the cycle repeats. Over many epochs the loss decreases, accuracy rises, and the network learns to extract meaningful features from the training images.

Civil Engineering Analogy

Training a CNN is like iterative structural design. You start with an initial design (random weights), run a structural analysis (forward pass), compare the results against code requirements (loss computation), identify which members are over- or under-stressed (backpropagation), and resize the members accordingly (weight update). You iterate this cycle until the design meets all criteria — that is convergence.

Step 1: The Forward Pass

The forward pass is the simplest step conceptually. An input image enters the network and flows through every layer in sequence: convolution, batch normalization, ReLU, pooling, and eventually the fully connected classifier. Each layer transforms its input and passes the result to the next layer.

Forward Pass Through CrackNet-Lite

Consider a 128×128 grayscale image of concrete entering the CrackNet-Lite architecture from Sub-chapter 4.4: 

  Image (128x128x1)
    |  Conv 3x3, 32 filters + BN + ReLU
    v
  Feature Maps (128x128x32)
    |  MaxPool 2x2
    v
  Feature Maps (64x64x32)
    |  Conv 3x3, 64 filters + BN + ReLU
    v
  Feature Maps (64x64x64)
    |  MaxPool 2x2
    v
  Feature Maps (32x32x64)
    |  Conv 3x3, 128 filters + BN + ReLU
    v
  Feature Maps (32x32x128)
    |  MaxPool 2x2
    v
  Feature Maps (16x16x128)
    |  Flatten
    v
  Vector (32768)
    |  Dense 256 + ReLU + Dropout
    v
  Vector (256)
    |  Dense 2 + Softmax
    v
  Probabilities: [P(crack), P(no_crack)]

At the end, the softmax function converts the raw output scores (logits) into probabilities that sum to 1. For a crack detection task, the output might be [0.87, 0.13] — meaning 87% confidence that the image contains a crack.

Crucially, during the forward pass every intermediate result is stored in memory. These cached activations are needed for backpropagation. This is why training requires significantly more GPU memory than inference (prediction only).

Memory Consideration

For CrackNet-Lite with a batch size of 32, the cached activations alone consume approximately 800 MB of GPU memory. Deeper architectures like ResNet-50 require several gigabytes. This is why batch size is constrained by available GPU memory — a concept we explore in detail later.

Step 2: Loss Computation

The loss function quantifies how wrong the network's predictions are. A perfect prediction yields a loss of zero; wildly incorrect predictions produce large loss values. The choice of loss function depends on the task:

Common Loss Functions

Loss Function Task Formula Intuition
Binary Cross-Entropy 2-class (crack / no crack) Penalizes confident wrong answers exponentially
Categorical Cross-Entropy Multi-class (crack type A/B/C) Extension of binary CE to N classes
Mean Squared Error Regression (predict deflection mm) Average squared difference between prediction and target

For our crack detection CNN, we use binary cross-entropy. If the true label is "crack" (1) and the network predicts probability p = 0.87, the loss for that sample is: 

  Loss = -[y * log(p) + (1-y) * log(1-p)]
       = -[1 * log(0.87) + 0 * log(0.13)]
       = -log(0.87)
       = 0.139

If the network had predicted p = 0.10 for the same crack image, the loss would be -log(0.10) = 2.303 — a much larger penalty. The loss function's shape ensures that the gradient pushes harder when the network is confidently wrong.

The batch loss is the average loss across all samples in the mini-batch. This averaged value is what backpropagation differentiates with respect to every weight in the network.

Step 3: Backpropagation

Backpropagation answers the question: "How much did each weight contribute to the total error?" It works by applying the chain rule of calculus, propagating the gradient of the loss backwards through every layer, from the output all the way to the first convolutional kernel.

Civil Engineering Analogy

Backpropagation is like tracing the cause of excessive deflection in a structure. You observe the deflection at a critical point (the loss). You trace backwards through the load path: which beam contributes to this deflection? Which column supports that beam? Which foundation element carries that column? Each element receives a "share of blame" proportional to its contribution. In a CNN, the chain rule performs exactly this backward tracing, assigning each weight its share of the error.

The Chain Rule in Action

Consider a simple chain of three operations: a convolutional layer, a ReLU activation, and a dense layer producing the loss. The gradient of the loss with respect to the convolutional weights is: 

  dL/dW_conv = dL/dOutput * dOutput/dReLU * dReLU/dConv * dConv/dW_conv
             = (chain of local gradients)

Each layer only needs to know two things: the gradient flowing in from the layer above (already computed), and its own local gradient (a simple mathematical expression specific to that layer type). Multiplying these gives the gradient to pass to the layer below.

Local Gradients for Common Layers

Layer Local Gradient
ReLU 1 if input > 0, else 0 (gradient gate)
Convolution Cross-correlation of gradient with rotated kernel
MaxPool Gradient passes only to the max element; others get 0
Dense (FC) Matrix multiply with transposed weights
Softmax + CE prediction - true_label (elegant simplification)

After backpropagation completes, every trainable parameter in the network has an associated gradient value. These gradients tell the optimizer exactly which direction and how far to adjust each weight.

Step 4: Weight Update & Optimizers

The optimizer uses the computed gradients to update the weights. The simplest approach is Stochastic Gradient Descent (SGD): 

  W_new = W_old - learning_rate * gradient

The learning rate controls the step size. Too large and the optimization overshoots and diverges; too small and training takes forever (or gets stuck in a local minimum).

Popular Optimizers

Optimizer Key Idea When to Use
SGD Simple gradient step Baseline; with momentum for most vision tasks
SGD + Momentum Accumulates velocity like a rolling ball Smoother convergence; standard in research
Adam Adaptive per-parameter learning rates Default choice; works well out of the box
AdamW Adam with decoupled weight decay Fine-tuning pretrained models

Adam is the most popular optimizer for training CNNs in practice. It maintains two running averages for each parameter: the mean of recent gradients (momentum) and the mean of recent squared gradients (adaptive scaling). This combination allows it to navigate flat regions quickly while taking careful steps near sharp loss landscape features.

Civil Engineering Analogy

Choosing an optimizer is like choosing an earthmoving strategy. SGD is a single bulldozer that pushes material downhill — effective but can get stuck in ruts. SGD with momentum is the same bulldozer with enough inertia to roll past small bumps. Adam is a smart fleet of machines that independently adapts speed and direction based on local terrain conditions.

Epochs, Batches, and Convergence

Understanding the relationship between epochs, batches, and iterations is essential for configuring training runs:

Terminology

  • Epoch — One complete pass through the entire training dataset.
  • Batch (mini-batch) — A subset of the dataset processed together in one forward/backward pass.
  • Iteration — One weight update step, corresponding to one batch.
  • Example: 10,000 images with batch size 32 = 313 iterations per epoch.

How Many Epochs?

There is no universal answer. For crack detection datasets of moderate size (5,000–20,000 images), typical training runs use 30–100 epochs. The key is to monitor the validation loss (loss measured on a held-out set of images the network has never seen):

  • Training loss decreasing, validation loss decreasing — the network is learning useful patterns. Continue training.
  • Training loss decreasing, validation loss increasing — overfitting. The network is memorizing training data rather than learning generalizable features. Stop training (or apply regularization).
  • Both losses plateaued — convergence reached. Further training will not help.

Early stopping is a common strategy: save the model weights whenever validation loss improves, and stop training when validation loss has not improved for a set number of epochs (the "patience" parameter).

Batch Size Effects

Batch size is one of the most impactful hyperparameters, affecting training speed, convergence behavior, and final model quality:

Small vs. Large Batch Sizes

Aspect Small Batch (8–32) Large Batch (128–512)
Gradient estimate Noisy (high variance) Smooth (low variance)
Convergence Can escape local minima May get stuck in sharp minima
GPU utilization Under-utilizes parallel hardware Maximizes GPU throughput
Memory required Lower (fits smaller GPUs) Higher (needs large VRAM)
Generalization Often better test accuracy Can generalize poorly

For civil engineering image classification, batch sizes of 16–64 typically offer the best balance. The moderate noise in gradient estimates acts as implicit regularization, helping the model generalize to new inspection images. A common starting point is batch size 32.

Linear Scaling Rule

When increasing batch size by a factor of k, multiply the learning rate by k as well. For example, if batch size doubles from 32 to 64, double the learning rate from 0.001 to 0.002. This heuristic maintains similar training dynamics across batch sizes.

Data Augmentation for Civil Engineering Images

In civil engineering, labeled datasets are expensive: each image requires expert annotation. Data augmentation artificially expands the dataset by applying random transformations to existing images during training. The network sees slightly different versions of each image every epoch, which reduces overfitting and improves generalization.

Augmentation Techniques for Structural Imagery

Technique Description CE Rationale
Horizontal Flip Mirror image left-right Cracks have no preferred horizontal direction
Vertical Flip Mirror image top-bottom Drone images can be oriented arbitrarily
Rotation (0–360) Rotate by random angle Cracks appear at any angle on a surface
Random Crop Extract a sub-region and resize Simulates varying camera distances
Brightness Jitter Randomly adjust brightness Inspections occur under varied lighting
Color Jitter Adjust contrast, saturation, hue Different cameras/weather produce color shifts
Gaussian Noise Add random pixel noise Simulates sensor noise in low-light conditions

Not all augmentations are appropriate for every task. For example, vertical flipping is safe for crack detection (cracks look the same upside-down) but would be harmful for tasks where gravity matters, such as detecting sagging beams.

Civil Engineering Analogy

Data augmentation is like testing a structure under many load combinations. Instead of testing only one dead load + one live load scenario, you apply dozens of combinations: dead + live + wind from the north, dead + live + seismic from the east, and so on. Each combination teaches you something new about the structure's behavior. Similarly, each augmented version of an image teaches the CNN to recognise cracks under a different "condition" (angle, brightness, scale).

Augmentation Multiplier

With a dataset of 5,000 labeled crack images and 7 augmentation techniques, each applied with 50% probability, the effective dataset size becomes approximately 5,000 × 27 = 640,000 unique image variants across all epochs. This massive expansion is why augmentation is considered the single most effective technique for small civil engineering datasets.

Transfer Learning: Standing on Giants' Shoulders

Training a CNN from scratch requires large datasets and days of GPU time. Transfer learning sidesteps both problems by starting from a model already trained on a massive dataset — typically ImageNet, which contains 1.2 million images across 1,000 categories.

Why Transfer Learning Works

The early layers of a CNN learn universal features — edges, textures, shapes — that are useful for almost any image task. A model trained on ImageNet already knows how to detect edges, gradients, and patterns. These features transfer directly to civil engineering imagery:

  • Edge detectors learned from cat photos also detect crack boundaries
  • Texture features learned from fabric images also characterize concrete grain
  • Shape detectors learned from cars also identify crack patterns

Two Transfer Learning Strategies

Strategy 1: Feature Extraction (Freeze + Retrain Head)

Load a pretrained model (e.g., ResNet-50), freeze all convolutional layers so their weights do not change, and replace the final classification head with a new one for your task (e.g., 2 outputs: crack / no crack). Only train the new head. 

  ImageNet-pretrained ResNet-50
    |
    [ Conv layers: FROZEN - weights unchanged ]
    |
    [ New Dense 256 + ReLU ]    <-- train only these
    [ New Dense 2 + Softmax ]   <-- train only these

Best when: your dataset is small (<1,000 images) and similar to ImageNet images.

Strategy 2: Fine-Tuning (Unfreeze + Retrain All)

Start with a pretrained model, replace the head, and train the entire network with a very small learning rate. The pretrained weights serve as a warm start rather than a fixed feature extractor. 

  ImageNet-pretrained ResNet-50
    |
    [ Conv layers: UNFROZEN - fine-tuned (lr=1e-5) ]
    |
    [ New Dense 256 + ReLU ]    (lr=1e-3)
    [ New Dense 2 + Softmax ]   (lr=1e-3)

Best when: your dataset is moderate (1,000–10,000 images) and somewhat different from ImageNet (e.g., infrared bridge images).

Civil Engineering Analogy

Transfer learning is like adaptive reuse of a building. Instead of demolishing an old warehouse and building from scratch, you keep the solid foundation and structural frame (pretrained convolutional layers) and renovate only the interior finishes and layout (the classifier head) for the new purpose. If the new use is very different from the original, you might also reinforce some structural members (fine-tuning), but you never throw away the foundation.

Typical Accuracy Comparison

Approach 2,000 images 10,000 images Training Time
From scratch 78% 91% 8–12 hours

The message is clear: always use transfer learning for civil engineering CNN projects. Training from scratch is only justified when your images are radically unlike natural photographs (e.g., thermal imaging, ground-penetrating radar scans) and even then, the pretrained low-level features often help.

Learning Rate Scheduling

A fixed learning rate is rarely optimal throughout training. In early epochs, a larger learning rate helps the network quickly move toward a good region of the loss landscape. In later epochs, a smaller rate enables fine-grained convergence without overshooting the minimum. Learning rate schedulers automate this reduction.

Popular Scheduling Strategies

Schedule Behavior Use Case
Step Decay Multiply LR by 0.1 every N epochs Simple, predictable drops
Cosine Annealing LR follows a cosine curve to near-zero Smooth decay; popular for fine-tuning
Reduce on Plateau Reduce LR when val loss stops improving Adaptive; no need to pre-set schedule
Warm-up + Decay Start low, ramp up, then decay Transfer learning; prevents initial shock

For transfer learning on crack detection models, cosine annealing with a warm-up period of 3–5 epochs is a robust default. The warm-up prevents the freshly initialized classification head from generating large gradients that corrupt the pretrained convolutional weights. 

  Typical fine-tuning schedule:

  Epochs 1-3:   warm-up  lr = 1e-6 -> 1e-4 (linear ramp)
  Epochs 4-50:  cosine   lr = 1e-4 -> 1e-7 (cosine decay)
  Total: 50 epochs, initial lr = 1e-4

Overfitting and Regularization

Overfitting is the most common failure mode when training CNNs on small civil engineering datasets. The network memorizes specific crack patterns, staining, or background textures in the training images rather than learning generalizable features. On new inspection images, such a model performs poorly.

Signs of Overfitting

  • Training accuracy > 99% but validation accuracy < 85%
  • Training loss near zero while validation loss increases
  • Model confident on training images but uncertain on new images

Regularization Toolkit

Techniques to Combat Overfitting

  1. Data Augmentation — artificially expand the dataset (covered above).
  2. Dropout — randomly disable neurons during training (typically 20–50% in FC layers), forcing the network to learn redundant representations.
  3. Weight Decay (L2 Regularization) — add a penalty proportional to the magnitude of weights, discouraging overly complex models.
  4. Early Stopping — halt training when validation loss starts increasing.
  5. Batch Normalization — normalizes activations, which has a mild regularizing effect.
  6. Transfer Learning — pretrained features are already generalizable, reducing the risk of overfitting on small datasets.

In practice, the most effective anti-overfitting recipe for civil engineering CNNs is: transfer learning + data augmentation + dropout + early stopping. This combination consistently achieves strong performance even with datasets as small as 1,000 labeled images.

Practical Training Recipe for Civil Engineers

Putting it all together, here is a step-by-step recipe for training a crack detection CNN that you can follow on any project:

The 10-Step Training Recipe

  1. Collect and label data — Gather at least 1,000 images per class. Have a domain expert verify labels.
  2. Split the dataset — 70% training, 15% validation, 15% test. Ensure no data leakage between splits.
  3. Choose a pretrained backbone — ResNet-50 or EfficientNet-B0 are excellent starting points.
  4. Replace the classification head — Add a new dense layer matching your number of classes.
  5. Define augmentation pipeline — Horizontal flip, rotation, brightness jitter, random crop.
  6. Set hyperparameters — Optimizer: Adam (or AdamW). Initial LR: 1e-4. Batch size: 32. Epochs: 50.
  7. Configure LR scheduler — Cosine annealing with 3-epoch warm-up.
  8. Train with early stopping — Patience of 10 epochs on validation loss.
  9. Evaluate on test set — Report accuracy, precision, recall, and F1-score.
  10. Iterate — Examine misclassified images, adjust augmentation, try different backbones. 
  # Pseudocode: Complete Training Pipeline
  model = load_pretrained("resnet50", weights="imagenet")
  model.replace_head(num_classes=2)

  augmentation = Compose([
      RandomHorizontalFlip(p=0.5),
      RandomVerticalFlip(p=0.5),
      RandomRotation(degrees=15),
      ColorJitter(brightness=0.2, contrast=0.2),
      RandomResizedCrop(size=224, scale=(0.8, 1.0)),
  ])

  optimizer = AdamW(model.parameters(), lr=1e-4, weight_decay=1e-4)
  scheduler = CosineAnnealingWarmRestarts(optimizer, T_0=50)
  criterion = BinaryCrossEntropy()

  for epoch in range(50):
      for batch_images, batch_labels in train_loader:
          batch_images = augmentation(batch_images)      # Step 0: augment
          predictions = model(batch_images)               # Step 1: forward
          loss = criterion(predictions, batch_labels)     # Step 2: loss
          loss.backward()                                 # Step 3: backprop
          optimizer.step()                                # Step 4: update
          optimizer.zero_grad()

      val_loss, val_acc = evaluate(model, val_loader)
      scheduler.step()
      early_stopping.check(val_loss, model)

Common Training Pitfalls

Even experienced practitioners encounter these issues. Recognizing them early saves hours of wasted GPU time:

Pitfalls and Solutions

Symptom Likely Cause Fix
Loss = NaN Learning rate too high Reduce LR by 10x; add gradient clipping
Loss stuck at ln(2) = 0.693 Model outputs random 50/50 (not learning) Check data loader, labels, input normalization
Train acc 99%, val acc 70% Severe overfitting More augmentation, dropout, or more data
Val acc oscillates wildly LR too high or batch too small Reduce LR; increase batch size
GPU out of memory Batch size or model too large Reduce batch size; use mixed precision
Training very slow Data loading bottleneck Use multiple data loader workers; prefetch

Key Takeaways

Summary

  • CNN training is a four-step loop: forward pass → loss computation → backpropagation → weight update, repeated over many epochs.
  • The loss function (typically cross-entropy) quantifies prediction error; backpropagation distributes blame via the chain rule.
  • Data augmentation (flip, rotate, crop, brightness) is essential for small civil engineering datasets.
  • Transfer learning from ImageNet-pretrained models dramatically reduces data requirements and training time for crack detection and similar tasks.
  • Learning rate scheduling (cosine annealing with warm-up) and early stopping are standard practice for reliable convergence.
  • Batch size affects gradient noise, generalization, and GPU memory usage; 16–64 is a good starting range.
  • Combat overfitting with the regularization quartet: augmentation, dropout, weight decay, and early stopping.

Civil Engineering Application: Training CNNs for Infrastructure

Training a crack classifier involves unique challenges that don't exist in generic computer vision:

Data Collection Strategy

Crack images are rare (class imbalance: ~95% intact, ~5% cracked). Solutions: data augmentation (rotation, flip, brightness jitter), oversampling cracks 10×, or focal loss (γ=2) to focus training on hard crack examples. Without this, the CNN just predicts "no crack" for everything.

Transfer Learning Advantage

Instead of training from scratch (needs 50K+ images), fine-tune ImageNet-pretrained ResNet on just 5,000 crack images. Freeze first 3 blocks, retrain last block + classifier head. Achieves 95%+ accuracy vs. 80% from scratch — edge filters transfer directly from natural images to concrete textures.

Monitoring Overfitting

Bridge inspection datasets are small (1,000-10,000 images per bridge type). Watch for val_loss diverging from train_loss after epoch ~30 — this means the CNN memorized your training cracks but can't generalize to new bridges. Use early stopping + dropout (0.5) to prevent this.

Knowledge Check

Test your understanding of CNN training with these four questions.

Question 1 of 4

During backpropagation, what does the gradient of the loss with respect to a specific weight represent?

Correct! The gradient tells us the direction and magnitude of change in the loss for a tiny change in that weight. This is exactly the information the optimizer needs to reduce the loss.
Not quite. The gradient is the partial derivative of the loss with respect to the weight — it indicates how much and in which direction the loss changes when the weight is perturbed.
Question 2 of 4

Why is data augmentation particularly important for civil engineering CNN projects?

Correct! Labeled civil engineering datasets are expensive to create and typically small. Data augmentation creates diverse training variants from existing images, dramatically reducing overfitting and improving generalization.
Not quite. Augmentation does not reduce resolution or eliminate labeling needs. Its primary value is artificially expanding small labeled datasets to prevent the network from memorizing specific images.
Question 3 of 4

In transfer learning, what is the key difference between "feature extraction" and "fine-tuning"?

Correct! In feature extraction, the pretrained convolutional layers are frozen and only the new classification head is trained. In fine-tuning, all layers are unfrozen and trained with a very small learning rate to adapt the pretrained features to the new domain.
Not quite. The key distinction is whether the pretrained convolutional weights are frozen (feature extraction) or updated with a small learning rate (fine-tuning). Both strategies start from pretrained weights.
Question 4 of 4

During training, the training loss is decreasing steadily but the validation loss starts increasing after epoch 20. What is happening and what should you do?

Correct! When training loss decreases but validation loss increases, the network is overfitting — memorizing training data rather than learning generalizable features. Early stopping reverts to the best checkpoint (lowest validation loss), and you should consider adding more augmentation or dropout.
Not quite. This is a classic overfitting pattern: the divergence between training and validation loss indicates the model is memorizing the training set. Early stopping at the best validation checkpoint is the immediate fix.
