Foundations

The Artificial Neuron

Every neural network - including CNNs - is built from artificial neurons. Each neuron takes multiple inputs, multiplies each by a weight, adds a bias, and passes the result through an activation function.

CE Analogy: The Structural Load Calculator

Think of a neuron as an engineer evaluating whether a beam will fail. Inputs are load, span, cross-section, material strength. Each input has a different importance (weight). The engineer sums the weighted factors, applies judgment (activation), and outputs a decision: safe or unsafe.

Activation Functions - Adding Nonlinearity

Without activation functions, a neural network is just linear algebra - it can only draw straight lines. Activation functions introduce nonlinearity, allowing networks to model curves, boundaries, and complex patterns.

Classic

Sigmoid

σ(x) = 1/(1+e⁻ˣ). Squashes output to [0,1]. Used in binary classification. Problem: gradients vanish for large values.

Most Popular

ReLU

f(x) = max(0, x). Dead simple, computationally fast. Default choice for hidden layers. Can "die" if inputs are always negative.

Smooth

Tanh

Output range [-1, 1]. Zero-centered, which helps gradient flow. Often used in RNN cells and normalization layers.

Output Layer

Softmax

Converts a vector of numbers into probabilities that sum to 1. Used in the final layer for multi-class classification.

Architecture

From Perceptron to Deep Networks

A single neuron (perceptron) can only separate data with a straight line. Stack neurons in layers and you get a deep neural network that can learn incredibly complex patterns.

1

Single Perceptron (1958)

One neuron. Can learn AND, OR gates. Cannot learn XOR. Like a single traffic signal - limited decisions.

2

Multi-Layer Perceptron (1986)

Multiple layers of neurons. Can learn XOR and more complex boundaries. Like a team of engineers reviewing different aspects of a project.

3

Deep Neural Networks (2010s)

Many layers (10-100+). Each layer learns progressively more abstract features. Like a hierarchical review process from junior to senior to partner.

Why "deep"? A network with more than 2 hidden layers is considered "deep." Depth allows the network to build hierarchical representations - early layers detect simple patterns (edges), middle layers combine them (shapes), and deep layers recognize complex objects (buildings, cracks).
Motivation

Why CNNs for Images?

A regular neural network treats every pixel independently. For a 256×256 color image, that's 196,608 inputs, each connected to every neuron in the first hidden layer. This is hugely wasteful because:

Spatial Structure Matters

Adjacent pixels are related. A crack is a pattern of dark pixels in a line. Fully-connected networks ignore this spatial relationship.

Features are Local

An edge is defined by a small patch of pixels, not the entire image. We need a way to detect local patterns regardless of where they appear.

Translation Invariance

A crack in the top-left looks the same as a crack in the bottom-right. The detector should work regardless of position.

CE Analogy: The Site Inspector's Magnifying Glass

A site inspector doesn't look at the entire building simultaneously. They use a magnifying glass, scanning small patches systematically - left to right, top to bottom. They look for the same patterns (cracks, rust, spalling) everywhere. That's exactly what a CNN's convolutional filter does: a small window that slides across the image, detecting the same features everywhere.

Knowledge Check #1

Why do CNNs use small sliding filters instead of connecting every pixel to every neuron?
A Because GPUs can only process small matrices
B Image features are local and should be detected regardless of position
C To make the training faster without improving results
Correct! CNNs exploit spatial locality and translation invariance - features like edges and textures are local patterns that should be detected everywhere in the image using shared weights.
The key insight is that image features (edges, textures, shapes) are local and appear at any position. Shared sliding filters exploit both properties efficiently.
Core Operation

The Convolution Operation

The convolution is the fundamental operation in a CNN. A small matrix called a kernel (or filter) slides across the input image. At each position, it computes an element-wise multiplication and sums the result - producing one number in the output feature map.

1

Place the Kernel on a Patch

A 3×3 kernel covers a 3×3 patch of the input image. Think of placing a template over a small area.

2

Element-wise Multiply

Each kernel value multiplies the corresponding pixel value. 9 multiplications for a 3×3 kernel.

3

Sum the Products

Add all 9 products to get one number. This is the output pixel in the feature map.

4

Slide and Repeat

Move the kernel one pixel to the right (stride=1) and repeat. Continue row by row until the entire image is covered.

What Different Kernels Detect

Horizontal Edges

Kernel: [[-1,-1,-1],[0,0,0],[1,1,1]]. Detects transitions from dark to light vertically - like the boundary between a beam and the sky.

Vertical Edges

Kernel: [[-1,0,1],[-1,0,1],[-1,0,1]]. Detects left-right transitions - column edges, wall boundaries.

Corner / Texture

More complex kernels detect corners, textures, and gradients. Learned automatically during training.

Architecture

CNN Architecture - Layer by Layer

A complete CNN stacks multiple types of layers to progressively extract higher-level features from the raw pixel input.

CE Analogy: Hierarchical Site Inspection

Layer 1 (Conv): Inspector looks at individual pixels → detects edges and color gradients.
Layer 2 (Conv): Combines edges → recognizes shapes (circles, rectangles).
Layer 3 (Conv): Combines shapes → identifies objects (window, door, crack).
Final (Dense): Combines objects → classifies the building condition (safe/damaged/critical).
Each layer builds on the previous, just like how an inspection report aggregates findings.

Operation

Pooling - Downsizing Feature Maps

After convolution, pooling layers reduce the spatial dimensions. The most common is Max Pooling: take a 2×2 window, keep only the maximum value, discard the rest. This halves width and height.

Max Pooling

Keep the strongest activation in each region. "Is there an edge somewhere in this 2×2 area?" - if yes, the max value preserves it.

Average Pooling

Take the average of values in the window. Smoother but may lose sharp features. Used in some architectures at the final layer.

CE Analogy: Survey Resolution Reduction

You have a 1m×1m grid survey of a construction site (1000×1000 points). For preliminary analysis, you downsample to 2m×2m by taking the maximum elevation in each 2×2 block. You lose fine detail but preserve the important features (hills, depressions) while making computation 4× faster. That's exactly what max pooling does to feature maps.

Max Pooling - See the Reduction 6×6 → 3×3 via 2×2 max pooling

Knowledge Check #2

What is the primary purpose of pooling layers in a CNN?
A To add more features to the image
B To detect edges in the input image
C To reduce spatial dimensions while preserving important features
Correct! Pooling downsamples feature maps, reducing computation and providing some translation invariance while keeping the strongest activations.
Pooling doesn't detect features (that's convolution) or add information. It reduces spatial dimensions while preserving the most important activations.
History

Famous CNN Architectures

The evolution of CNN architectures is a story of going deeper, wider, and smarter. Each breakthrough pushed the boundaries of what machines can see.

1998

LeNet-5

Yann LeCun's pioneer. 5 layers, designed for handwritten digit recognition. Proved CNNs work. Like the first theodolite - crude but revolutionary.

2012

AlexNet

8 layers, GPU-trained. Won ImageNet by a landslide, igniting the deep learning revolution. Used ReLU activation and dropout. The "aha" moment.

2014

VGGNet

16-19 layers of uniform 3×3 convolutions. Showed that depth matters. Simple but very effective. Still widely used as a feature extractor.

2014

GoogLeNet / Inception

22 layers with "inception modules" - parallel convolutions at different scales. More efficient than VGG. 12× fewer parameters.

2015

ResNet

152 layers with "skip connections" (residual learning). Solved the vanishing gradient problem. Enabled truly deep networks. Won ImageNet 2015.

2020+

EfficientNet / Vision Transformers

Modern architectures balance width, depth, and resolution. Vision Transformers (ViT) apply the Transformer architecture to images, rivaling CNNs.

ResNet's Key Innovation: Skip connections let gradients flow directly through the network, bypassing layers. This is like an express elevator in a skyscraper - instead of climbing every floor (layer), information can skip ahead, preventing it from degrading.
Applications

CNNs in Civil Engineering

Most Common

Crack Detection

CNNs analyze drone/camera images of concrete surfaces, automatically detecting and classifying cracks by severity. Replaces manual visual inspection.

Remote Sensing

Land Cover Classification

Classify satellite imagery pixels into urban, forest, water, agriculture classes. U-Net architecture is popular for semantic segmentation of terrain.

Structural

Damage Assessment

Post-earthquake/disaster: CNNs classify building damage from aerial imagery as none/minor/major/destroyed. Speeds up response by days.

Transportation

Pavement Condition

Vehicle-mounted cameras feed images to CNNs that detect potholes, rutting, raveling, and alligator cracking automatically.

Safety

PPE Compliance

Real-time detection of workers without helmets, vests, or safety gear on construction sites using object detection CNNs (YOLO, SSD).

Geotechnical

Soil Classification from Images

Classify soil type from borehole images or thin-section microscopy. CNN learns texture patterns that correlate with engineering properties.

Code Peek

Building a CNN in Code

Crack Detection CNN (PyTorch)

crack_detector.py 
import torch.nn as nn

class CrackDetector(nn.Module):
    def __init__(self):
        super().__init__()

        # Layer 1: 3 input channels (RGB) → 32 feature maps
        # 3×3 kernel slides across the image detecting edges
        self.conv1 = nn.Conv2d(3, 32, kernel_size=3, padding=1)
        self.relu1 = nn.ReLU()
        self.pool1 = nn.MaxPool2d(2) # 224×224 → 112×112

        # Layer 2: 32 → 64 feature maps (detects shapes)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.relu2 = nn.ReLU()
        self.pool2 = nn.MaxPool2d(2) # 112×112 → 56×56

        # Layer 3: 64 → 128 feature maps (detects crack patterns)
        self.conv3 = nn.Conv2d(64, 128, kernel_size=3, padding=1)
        self.relu3 = nn.ReLU()
        self.pool3 = nn.MaxPool2d(2) # 56×56 → 28×28

        # Flatten and classify
        self.flatten = nn.Flatten()
        self.fc1 = nn.Linear(128 * 28 * 28, 256)
        self.fc2 = nn.Linear(256, 2) # 2 classes: crack / no crack

    def forward(self, x):
        x = self.pool1(self.relu1(self.conv1(x))) # edges
        x = self.pool2(self.relu2(self.conv2(x))) # shapes
        x = self.pool3(self.relu3(self.conv3(x))) # patterns
        x = self.flatten(x)
        x = nn.ReLU()(self.fc1(x))
        return self.fc2(x) # [crack_score, no_crack_score]
Notice the pattern: Conv → ReLU → Pool repeated 3 times, then Flatten → Dense → Output. This is the classic CNN architecture. Each conv layer doubles the number of filters (32→64→128) while pooling halves the spatial dimensions (224→112→56→28).

Using a Pre-trained Model (Transfer Learning)

transfer_learning.py 
import torchvision.models as models

# Load ResNet-50 pre-trained on ImageNet (14M images, 1000 classes)
model = models.resnet50(pretrained=True)

# Freeze all layers - don't retrain what's already learned
for param in model.parameters():
    param.requires_grad = False

# Replace only the final layer for our 4-class problem:
# [no_damage, minor_crack, major_crack, spalling]
model.fc = nn.Linear(2048, 4)

# Now train on just 500 labeled concrete images
# The pre-trained layers already know edges, shapes, textures!
# We only teach the final layer to map those features to our classes
Transfer Learning is incredibly powerful in civil engineering because we often have small datasets. Instead of training a CNN from scratch (needs millions of images), we reuse a model trained on ImageNet and only fine-tune the last layer. The early layers already know how to detect edges and textures - that knowledge transfers to crack detection, damage assessment, etc.
Assessment

Chapter Summary & Final Quiz

Convolution

Small kernels slide across images detecting local features (edges, textures). Shared weights enable translation invariance.

Hierarchical Features

Early layers → edges. Middle layers → shapes. Deep layers → objects. Each layer builds on the previous.

Key Architectures

LeNet → AlexNet → VGG → ResNet. Skip connections solved deep training. Transfer learning enables small-dataset applications.

CE Applications

Crack detection, damage assessment, land cover classification, PPE compliance, pavement monitoring - all powered by CNNs.

Final Challenge

You need to build a CNN to detect pavement cracks from camera images, but you only have 200 labeled images. What's the best approach?
A Train a 50-layer CNN from scratch on the 200 images
B Use transfer learning: load a pre-trained ResNet and fine-tune the last layer
C Use K-Means clustering on the pixel values
D Apply a fixed Sobel edge detection filter without any ML
Excellent! Transfer learning is the standard approach when you have limited data. A pre-trained ResNet already understands edges, textures, and shapes from millions of images. You just teach the final layer to map those features to "crack" vs "no crack."
Training from scratch with only 200 images would massively overfit. K-Means is unsupervised and pixel-level, not image-level. Sobel is a fixed filter, not learned. Transfer learning leverages pre-trained knowledge and works great with small datasets.
